

音质达- Nike 供应链项目 分工是单人跟整个项目的模式，需要处理ETL

- 数据基础
	- global有健全的 数仓体系 Teradata/Snowflake ，有仓库业务相关数据
	- 供应商（宝尊电商）通过API 提供一些电商相关数据，例如退货等
	- 为了满足报表要求，中国业务部自己维护了一个 数据库 Mysql 供报表使用
- 主要职能包括
	- 通过 Python 接口拉取数据存储本地
	- 使用 Kettle 调用 存储过程，将需要的数据拉取到 Mysql 
	- Mysql 中整合相关数据，给报表用
	- 使用 Windows Server 调度管理 Kettle 执行


数聚 - 通用磨坊项目

- 项目模式是 敏捷迭代式开发
- 早期并没有 ETL 同事参与，需要自己整合相关逻辑，在自定义sql 环节处理逻辑
- 后期，相关 ETL 同事落表，建立调度
